# コンテンツ設計

## コンテンツソース
| ソース | 用途 | ライセンス |
|--------|------|-----------|
| MS Learn | ベース情報。正確な仕様の根拠 | CC BY 4.0（出典明記で利用可） |
| MS公式ドキュメント | SLA、上限値、料金比較など具体的数値 | 同上 |
| 過去問・模擬問題の傾向 | 出題ポイントの把握、判断軸の整理 | 問題文そのものは掲載しない |

## コンテンツ方針
- MS Learnの文章をそのままコピペしない。自分の言葉で再構成する
- 長い説明文より**比較表**と**判断フロー**を重視する
- 各ページに出典としてMS Learnへのリンクを記載する
- 試験で実際に問われるポイントを明示する

## 参考書ページのテンプレート

各ページは以下の構造で統一する：

```
# [トピック名]

## 概要
このサービス/機能は何か。一言で説明。

## いつ使うか
どのような要件のときにこのサービスを選ぶのか。

## 主要な機能・特徴
箇条書きまたは表で整理。

## 他サービスとの比較
「AとBどちらを選ぶか」の判断軸を表で整理。
試験で最も問われる部分。

## 試験での出題ポイント
過去問の傾向から、特に注意すべき点。

## 出典
- [MS Learn: ○○](URL)
- [公式ドキュメント: ○○](URL)
```

## 4択問題のデータ形式

```json
{
  "topic": "entra-id",
  "questions": [
    {
      "id": 1,
      "question": "問題文",
      "choices": [
        "A. 選択肢1",
        "B. 選択肢2",
        "C. 選択肢3",
        "D. 選択肢4"
      ],
      "answer": 0,
      "explanation": "解説文。なぜこの選択肢が正解で、他が不正解なのかを説明。",
      "reference": "entra-id.html#関連セクション"
    }
  ]
}
```
